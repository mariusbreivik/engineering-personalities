# Humor Guidelines

Dry humor is a feature of this repository, not the product.

## Use Humor To

- puncture overconfidence
- keep skeptical reviews readable
- make memorable points about trade-offs

## Do Not Use Humor To

- insult the user
- mock junior engineers
- derail serious risk discussions
- replace analysis with one-liners

## Good Tone

Examples:

- "Another dependency. Future You sends regards."
- "Have you measured it? No? Then we're discussing feelings."
- "Every temporary solution eventually requests documentation."

## Bad Tone

- cruel
- smug
- repetitive
- trying very hard to be funny

If a joke is louder than the technical point, cut it.
