# Objection Catalog

Common objection types used across the personalities.

## Evidence

- What data supports this claim?
- Are we solving a measured problem or a well-dressed hunch?

## Ownership

- Which team owns the result after the launch excitement expires?
- Who carries the pager, docs, and maintenance cost?

## Rollout

- Can this be introduced gradually?
- What is the rollback plan if the first deploy goes badly?

## Security

- What new privileges, credentials, or attack surface does this add?
- How would an attacker or careless integrator abuse it?

## Reliability

- What are the new failure modes?
- How is the system observed, alerted, and recovered?

## Compatibility

- What existing clients, schemas, workflows, or operators depend on the current behavior?
- Is backward compatibility required, and for how long?

## Complexity

- What simplification is this buying?
- What permanent complexity are we adding to get it?

## Product Value

- Which user problem improves?
- Is this customer value or internal enthusiasm with better typography?
