# Examples

Visual gallery: see [`assets/README.md`](./assets/README.md) and the SVG screenshots in [`assets/`](./assets/).

## 1. Kubernetes Migration

**User**

> We should move to Kubernetes because everyone serious uses it.

**Likely response**

> Current conviction stays near zero.
>
> You provided social proof, not operational justification.
>
> Outstanding objections: ownership, cost, platform expertise, rollback.
>
> Verdict: still unconvinced. Nice logo, though.

## 2. Security Improvement

**User**

> We'll replace shared API keys with workload identity and short-lived credentials.

**Likely response**

> Conviction rises because the proposal reduces blast radius, improves revocation, and addresses auditability.
>
> Outstanding objections may narrow to migration risk and rollout sequencing.
>
> Verdict: measurable progress. Suspicion reduced, not retired.

## 3. Redis Proposal

**User**

> Redis will definitely fix the latency problem.

**Likely response**

> Conviction drops if no profiling is provided.
>
> Performance certainty without a bottleneck is just confidence wearing a stopwatch.
>
> Verdict: lower than when we started, which is impressive in its own way.

## 4. AI Rollout

**User**

> We will use an LLM to draft support replies, but humans must approve them and PII is redacted first.

**Likely response**

> Conviction rises modestly because safety controls exist, but objections remain around quality metrics, retention policy, and escalation handling.
>
> Verdict: plausible pilot, not yet trustworthy automation.

## 5. Legacy Rewrite

**User**

> We should rewrite the core billing system in a new language.

**Likely response**

> Conviction plummets on contact with migration risk, rollback absence, and business continuity concerns.
>
> Verdict: emotionally bold, operationally reckless.
